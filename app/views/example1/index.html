Versioning

The ability to create multiple independent versions of an iterative prototype.

Creating independent version of your prototype can help your to prove how you've iterated your prototype based on UR insights (Service Standard 8).

Start with a folder structure

Inside your app > views folder create a folder that you'll store your files in. Remember what you name the folder as you'll need it in reference it later.

Here's an example for this project https://github.com/MPK-13/prototyping-examples/tree/main/app/views (opens in a new window)

Create independent routes

Inside your version folder create a new routes file for that version. I call them _routes.js so I don't confuse it with the master routes file.

You should also create the variable version to save time later. Just remember to include the version variable in you redirects so the server know which version you're talking about.

Here's an example for this project https://github.com/MPK-13/prototyping-examples/tree/main/app/views/example1/_routes.js (opens in a new window)

Link the routes files

The master routes file, app > routes, is then used to point to all of your individual routes files

Here's an example for this project https://github.com/MPK-13/prototyping-examples/tree/main/app/routes.js (opens in a new window)

Creating a new version

Once this is all setup, to create a new version you need to:

  1. Duplicate your version folder and give it a new name
  2. Change the version variable in the _routes.js file you've just created as part of the folder duplication.
    var version = 'this should be your new folder name'
  3. Duplicate the require('./views/example1/_routes') line in the main routes file and point it at your new version.
